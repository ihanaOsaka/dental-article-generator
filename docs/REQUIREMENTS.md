# MeshiPick（メシピック） - フードサジェストアプリ 要件定義書

## 1. アプリ概要

**コンセプト**: 「今日何食べる？」に迷った時、あなたの食事履歴・お気に入りのお店・レシピから最適なごはんをサジェストしてくれるアプリ

**キーワード**: 簡単入力 / パーソナライズ / 外食 & 自炊 両対応

---

## 2. ターゲットユーザー

- **Phase 1**: 個人利用（開発者自身）
- **Phase 2**: 一般公開（将来的に）

---

## 3. 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | Next.js 14 (App Router) + TypeScript | SSR/SSG対応、PWAでモバイル対応 |
| UI | Tailwind CSS + shadcn/ui | 高速なUI構築、モバイルファースト |
| PWA | next-pwa | オフライン対応、ホーム画面追加、カメラアクセス |
| バックエンド | Next.js API Routes | フロントと統合、シンプルな構成 |
| データベース | Supabase (PostgreSQL) | 認証・DB・ストレージ一体型、無料枠あり |
| 画像ストレージ | Supabase Storage | 料理写真の保存 |
| 地図API | Google Maps Platform (Places API, Maps JavaScript API) | お店の検索・ブックマーク連携 |
| AI/アルゴリズム | ルールベース → 将来的にAI | まずはシンプルなロジックから |
| デプロイ | Vercel | Next.jsと最適な相性 |

## 4. コア機能

### 4.1 食事サジェスト機能 🎯（メイン機能）

ユーザーの以下の情報からアルゴリズムでサジェスト：
- 食事履歴（最近何を食べたか）
- よく行くお店 / ブックマークしたお店
- 保存済みレシピ
- 曜日・時間帯・天候などのコンテキスト

**サジェストのロジック（MVP）**:
- 直近3日間で食べていないジャンル/料理を優先
- よく行くお店の中から距離が近いものを優先
- 外食 or 自炊の傾向に合わせて提案
- ランダム要素を加えて飽きさせない

### 4.2 食事記録機能 📝

**「昨日なに食べた？」方式の簡単入力**:
- 対話形式で食事を記録
  - 「お昼は何を食べましたか？」→ 入力
  - 「外食？おうちごはん？」→ ワンタップ選択
  - 「お店の名前は？」→ Google Places検索 or スキップ
- 写真撮影 / アルバムから選択（任意）
- 食事カテゴリの自動推定（写真 or テキストから）

**記録する情報**:
| 項目 | 入力方法 | 必須/任意 |
|------|---------|----------|
| 日付・時間帯（朝/昼/夜） | 自動 + 手動修正 | 必須 |
| 料理名 | テキスト入力 / 写真から推定 | 必須 |
| 外食 or 自炊 | ワンタップ切り替え | 必須 |
| お店名 | Google Places検索 | 任意（外食時） |
| 写真 | カメラ / アルバム | 任意 |
| メモ / 感想 | テキスト | 任意 |
| 評価 | 5段階 or いいね | 任意 |

### 4.3 お店管理機能 🏪

**Google Maps連携**:
- Google アカウントでログイン
- ブックマーク（保存済みの場所）の取得
- よく行くお店の自動検出（履歴ベース）
- お店の詳細情報取得（ジャンル、営業時間、評価など）

**アプリ内管理**:
- お気に入りお店リスト
- お店ごとのメモ・評価
- ジャンルタグ付け

### 4.4 レシピ管理機能 🍳

**レシピの保存方法**:
- URL保存（外部レシピサイトのリンク）→ OGP情報を自動取得
- テキスト入力（自分のレシピを記録）
- 写真付きレシピ作成

**レシピの属性**:
- 料理名
- カテゴリ/ジャンル
- 所要時間（目安）
- 材料リスト
- 手順
- 写真
- 公開/非公開フラグ

**レシピの公開**:
- 公開設定にしたレシピは共有URL発行
- 将来的にレシピコミュニティ機能も検討

## 6. 入力の簡略化（UXの工夫）

このアプリの成否は **入力のハードルをどこまで下げられるか** にかかっている。

### 6.1 対話形式入力
- 一問一答形式で負担を軽減
- スキップ可能な項目は明示
- 前回の回答を元にデフォルト値をセット

### 6.2 写真ベース入力
- 写真を撮るだけで料理名を自動推定（将来的にAI）
- 外食の場合、位置情報からお店を自動推定

### 6.3 クイック入力
- 「よく食べるもの」リストからワンタップ登録
- 直近の食事を「また食べた」でコピー登録
- 音声入力対応（ブラウザのSpeech API）

### 6.4 リマインダー
- PWA通知で「お昼何食べた？」と聞いてくれる
- 記録忘れ防止

## 7. データモデル（概要）

```
User
├── id, email, name, avatar
├── google_account_linked: boolean
│
├── MealLog（食事記録）
│   ├── id, date, meal_type (朝/昼/夜/間食)
│   ├── dish_name, category
│   ├── is_eating_out: boolean
│   ├── restaurant_id (nullable)
│   ├── photo_url (nullable)
│   ├── rating, memo
│   └── created_at
│
├── Restaurant（お店）
│   ├── id, google_place_id
│   ├── name, address, category
│   ├── lat, lng
│   ├── is_favorite: boolean
│   ├── user_rating, user_memo
│   └── visit_count
│
├── Recipe（レシピ）
│   ├── id, title, category
│   ├── source_url (nullable)
│   ├── ingredients, steps
│   ├── cooking_time, photo_url
│   ├── is_public: boolean
│   └── created_at
│
└── SuggestionHistory（サジェスト履歴）
    ├── id, suggested_at
    ├── suggestion_type (外食/自炊)
    ├── suggested_item (料理名 or お店名)
    ├── was_accepted: boolean
    └── feedback
```

## 1. コンセプト

モンハンのオトモアイルーのように、**AIの3D相棒キャラクター「アイル」がリアルタイムでゲーム画面を見ながら掛け合いをする**仕組み。

```
┌─────────────────────────────────────────────────────────┐
│                      あなたのPC                           │
│                                                           │
│  ┌──────────┐   画面キャプチャ(1fps)  ┌───────────────┐  │
│  │ モンハン   │ ─────────────────────→ │ Python中継     │  │
│  │ ワイルズ   │                        │ サーバー       │  │
│  └──────────┘                        │               │  │
│                                       │ マイク音声 ──→│  │
│  ┌──────────┐   AI音声(Virtual Cable)  │               │  │
│  │ VSeeFace  │ ←──────────────────── │   WebSocket   │  │
│  │(3Dアバター) │                        └───────┬───────┘  │
│  └─────┬────┘                                │          │
│        │ ウィンドウキャプチャ                      │          │
│        ▼                                     ▼          │
│  ┌──────────┐                     Gemini Live API       │
│  │   OBS     │ ← ゲーム画面                (Google)      │
│  │  (合成)    │                                           │
│  └─────┬────┘                                           │
│        ▼                                                 │
│   YouTube / 録画ファイル                                    │
└─────────────────────────────────────────────────────────┘
```

## 2. 技術選定：なぜGemini Live APIか

### API比較

| 観点 | Gemini Live API | OpenAI Realtime API |
|------|----------------|-------------------|
| **映像入力** | ネイティブビデオストリーム（1fps） | スナップショット方式のみ |
| **音声レイテンシ** | 1.2〜2.3秒 | 0.45〜2.0秒 |
| **セッション時間** | 圧縮+再開で実質無制限 | 最大60分（再接続で延長） |
| **コスト** | 同等ティアで20〜50%安 | やや高い |
| **無料枠** | 1日1,000リクエスト | 限定的 |
| **ゲーミング用途** | 公式デモアプリあり | なし |
| **日本語** | 対応済み（70言語以上） | 対応済み |

**→ Gemini Live API一択。** Google公式がゲーミングアシスタントのデモを提供しており、まさにこのユースケースを想定している。

### 公式ゲーミングアシスタントデモ

Googleが公式に **Gaming Assistant Demo App** を公開している。これをベースに改造するのが最速。

- **リポジトリ:** GoogleCloudPlatform/generative-ai → gaming-assistant-demo-app
- **機能:**
  - 画面共有でゲームプレイを視聴
  - 音声コメンタリーも聴取
  - ペルソナ切替（Wise Wizard、SciFi Robot、Commander）
  - プロアクティブアシスタンス（ユーザーが助けを求めたとき自発的に発話）
  - Native Audio対応で低レイテンシの表現力豊かな音声

## 3. Gemini Live API 詳細仕様

### 接続

| 項目 | 内容 |
|------|------|
| プロトコル | WebSocket（ステートフル） |
| SDK | `google.genai` パッケージ（Python） |
| 接続方法 | `client.aio.live.connect()` |
| 推奨構成 | Python(FastAPI)バックエンド + フロントエンド |

### モデル

| モデル | 用途 | 状態 |
|--------|------|------|
| gemini-live-2.5-flash-native-audio | **推奨。** 低レイテンシ双方向音声映像 | GA |
| gemini-2.0-flash | 旧モデル | **2026/6/1 廃止予定** |
| gemini-3-flash (Preview) | 次世代 | Preview |

### 音声仕様

| 項目 | 値 |
|------|------|
| 送信サンプルレート | 16,000 Hz |
| 受信サンプルレート | 24,000 Hz |
| フォーマット | 16-bit PCM (paInt16) |
| チャンネル | モノラル |
| チャンクサイズ | 1024 フレーム |

### プリビルトボイス（8種）

| 名前 | 特徴（推定） |
|------|-------------|
| Puck | デフォルト |
| Charon | — |
| Kore | — |
| Fenrir | — |
| Aoede | — |
| Leda | — |
| Orus | — |
| Zephyr | — |

※ カスタムボイスは現時点で非対応。「アイル」のキャラクター性はボイス選択+システムプロンプトで制御。

### トークン消費

| 入力タイプ | 消費量 |
|-----------|--------|
| 音声（入力/出力） | 25 tokens/秒 |
| 映像（入力、1fps固定） | 258 tokens/秒 |
| **音声+映像合計** | **約283 tokens/秒** |

### セッション管理（最重要）

| 設定 | 音声のみ | 音声+映像 |
|------|---------|----------|
| 圧縮なし | 最大15分 | **最大2分** |
| コンテキスト圧縮有効 | **無制限** | **無制限** |

- コンテキストウィンドウ: **128Kトークン**
- 圧縮なしの場合: 128K / 283 ≒ **約7.5分** でコンテキストが埋まる
- **コンテキストウィンドウ圧縮は必須。** サーバー側スライディングウィンドウで古いターンを自動切り捨て
- WebSocket接続は約10分で切断される場合あり → **セッション再開（Session Resumption）機能で再接続**
- 再開トークンの有効期限: **2時間**（Vertex AI）

**実装上のポイント:**
```
セッション開始 → 圧縮有効化 → プレイ中は自動圧縮
→ WebSocket切断検知 → 再開トークンで自動再接続
→ 2時間以上の中断がなければシームレスに継続
```

## 4. 3Dアバター（VRM）構成

### ツール比較

| ツール | 特徴 | 音声リップシンク | コスト |
|--------|------|----------------|--------|
| **VSeeFace（推奨）** | 最も情報豊富。A/I/U/E/Oブレンドシェイプ+拡張10種対応 | ◎ マイク入力で駆動可能 | 無料 |
| **VMagicMirror** | Webカメラ不要。キーボード/マウス連動 | ◯ | 無料(Standard)/有料(Full) |
| **Warudo（高機能）** | 音声+モーキャプのリップシンク同時併用可能（VSeeFaceでは不可） | ◎ | 有料 |
| **3tene** | 初心者向けUI。Steam評価86% | ◯ | 無料/PRO有料 |

**→ VSeeFaceが最適。** AI音声入力デバイスを自由に選択でき、Virtual Cableを「マイク」指定でリップシンク可能。情報量も最多でトラブルシューティングしやすい。

### 必要ソフトウェア（すべて無料）

| ツール | 用途 |
|--------|------|
| **VRoid Studio** | 3Dキャラクター作成 → VRM出力 |
| **VSeeFace** | VRMモデル表示 + 音声リップシンク |
| **VB-Audio Virtual Cable** | 仮想オーディオデバイス |
| **VoiceMeeter Banana** | オーディオミキシング（複数出力先への分配） |
| **OBS Studio** | ゲーム画面 + アバター合成 |

### オーディオルーティング（詳細）

```
Gemini音声出力（Pythonスクリプト）
    │
    ↓ 出力先: 「CABLE Input (VB-Audio Virtual Cable)」
    │
VB-Audio Virtual Cable
    │
    ↓ CABLE Output
    │
VoiceMeeter Banana（HW Input 3 に CABLE Output を接続）
    │
    ├──→ A1: スピーカー/ヘッドフォン（あなたにも聞こえる）
    ├──→ B1: OBS（配信音声として取り込み）
    └──→ VSeeFace（マイク入力に「CABLE Output」を指定 → リップシンク駆動）
```

**VSeeFace側の設定:**
- マイク/音声入力デバイス → 「CABLE Output」を選択
- Hybrid lipsync → ON
- Viseme smoothing → 低めに設定（レスポンス重視）
- Audio gain overdrive → 0

**遅延:** Virtual Cable自体はカーネルレベルのルーティングでほぼゼロ遅延。VSeeFaceの音声→ブレンドシェイプ変換は数十ms。**体感的にはほぼリアルタイム**で口が動く。

### OBS合成設定

**推奨方式: Game Capture + 透過（クロマキー不要）**

| 方式 | メリット | デメリット |
|------|---------|-----------|
| **Game Capture + 透過（推奨）** | 設定が簡単、色被りなし、高画質 | Windows限定 |
| クロマキー | macOS等でも使える | 緑色衣装に干渉、エッジノイズ |
| Spout2 | 高品質、低遅延 | プラグイン要 |

**手順:**
1. OBSで「ソース」→「ゲームキャプチャ」→ VSeeFaceを選択
2. **「透過を許可」(Allow transparency) にチェック** → 背景が自動で透明に
3. ゲーム画面のキャプチャソースを追加
4. ソース順序: アバターをゲーム画面より**上**に配置
5. 配置: 画面右下 or 左下に10〜20%程度のサイズ
6. 表示範囲: 肩から上のみが一般的（手の動きが不自然になりやすいため）

### VRMモデル作成・入手

**VRoid Studio**（無料、Steam評価94%）で作成が最も手軽。

- 最新版 v2.11.0（2026年2月）。髪型プリセット、テクスチャエディターなど充実
- Windows / macOS / iPad対応

**既製モデルの入手先:**

| サービス | 内容 |
|---------|------|
| **VRoid Hub** | pixiv運営。VRMモデルの閲覧・DL |
| **BOOTH** | 無料VRoidモデル3,284件以上。有料オリジナルも多数 |
| **Live3D** | 100以上のモデルDL可能 |

**オリジナルキャラ（アイルー風など）の外注費用目安:**

| 依頼先 | 費用 | 期間 |
|--------|------|------|
| VRoidベース（個人クリエイター） | 1万〜10万円 | 約1ヶ月 |
| フルスクラッチ（個人） | 10万〜80万円 | 約3ヶ月 |
| クラウドソーシング | 5万〜30万円 | 30〜60日 |

※ VRoid Studioは人型特化のため、アイルー風の非人型キャラには向かない。
※ Phase 2では**まずVRoidで人型の仮モデルを作成**し、本番モデルは後から差し替える方が効率的。

## 6. 技術的リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| ゲーム画面の理解精度が低い | 的外れなコメント連発 | **むしろコンテンツになる。** 「お前何見てんだよ」→ 笑い |
| セッション切断 | 会話が途切れる | セッション再開機能で自動再接続。録画なら編集でカバー |
| レイテンシ1〜2秒 | 戦闘中のアドバイスが間に合わない | 戦闘中は短い感嘆詞のみ。詳細アドバイスは戦闘後に |
| コスト超過 | 長時間プレイで費用増 | トークン監視 + 映像送信の一時停止機能 |
| 映像1fps固定 | 素早い動きを捉えられない | モンハンの戦闘は比較的読めるペース。問題は軽微 |
| カスタムボイス非対応 | キャラクター性の限界 | 8種のプリビルトから最適なものを選択。将来的に対応待ち |
| Gemini 2.0 Flash廃止 | 2026/6月以降使えない | 2.5 Flash（GA済み）に移行。互換性あり |

## 7. 先行事例：AI VTuber / AI + ゲーム配信

### Neuro-sama（世界最大のAI VTuber）

| 項目 | 内容 |
|------|------|
| 開発者 | Vedal987（プログラマー） |
| 規模 | Twitchアクティブサブスクライバー16.2万以上（2026年1月） |
| LLM | 2Bパラメータの独自トランスフォーマーモデル |
| ゲームAI | 80x60pxグレースケール画面画像入力。osu!/Minecraftをプレイ |
| アバター | Live2D（2024年12月に3代目）、2025年11月に3Dモデル導入 |
| 記憶 | 約1週間の出来事を記憶可能 |

→ **AI + ゲーム配信が成立することの証明。** ただしNeuro-samaは「AIが自律プレイ」、当チャンネルは「人間がプレイ＋AIが相棒」という異なる構造。

### 注目のオープンソースプロジェクト

| プロジェクト | 特徴 |
|------------|------|
| **Realtime Avatar AI Companion** | **Google Gemini搭載。** VRMアニメキャラ、音声クローン、リアルタイムリップシンク。Google Colabで無料実行可 |
| **AI-Waifu-Vtuber** | YouTube/Twitch配信用。Virtual Cable経由でVTubeStudioとリップシンク連携。OBSテキストでリアルタイム字幕 |
| **Airi (moeru-ai)** | セルフホスト型AIコンパニオン。リアルタイム音声チャット、Minecraft/Factorioプレイ対応 |
| **ProjectBEA** | モジュール式AI VTuberエンジン。RAGメモリ、OBS連携、自律Minecraftエージェント |

→ **特に「Realtime Avatar AI Companion」はGemini + VRM + リップシンクの構成で、当プロジェクトに最も近い。** 参考コードとして活用可能。

### 典型的なAI VTuber技術スタック

```
マイク/チャット → STT(音声認識) → LLM(応答生成) → TTS(音声合成)
                                                        ↓
                                            Virtual Cable → VSeeFace(リップシンク)
                                                        ↓
                                                   OBS → 配信
```

当プロジェクトではGemini Live APIがSTT + LLM + TTSを一気通貫で処理するため、この構成がシンプルになる。
